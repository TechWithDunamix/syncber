Welcome to the **Cloud Engineering** course — your complete path to designing and managing cloud infrastructure across AWS, Azure, and GCP.

<Aside variant="info" title="Course Format">
  Self-paced · 12 modules · 4 phases · Hands-on labs, real-world architecture projects, and infrastructure-as-code exercises included.
</Aside>

---

## What You'll Learn

<Steps>
1. **Phase 1: Cloud Foundations** — Cloud concepts, networking, storage, and database services
2. **Phase 2: AWS Deep Dive** — Core AWS services, serverless, containers, and architecture best practices
3. **Phase 3: Multi-Cloud & IaC** — Microsoft Azure, Google Cloud Platform, Terraform, and CloudFormation
4. **Phase 4: Cloud Operations** — Monitoring, observability, security, compliance, and a capstone project
</Steps>

---

## Course Structure

| Phase | Modules | Topics Covered |
|-------|---------|---------------|
| **1. Cloud Foundations** | Cloud Fundamentals, Cloud Networking, Storage & Databases | Cloud models, deployment options, global infrastructure, VPCs, subnets, DNS, CDN, load balancing, S3, EBS, RDS, DynamoDB |
| **2. AWS Deep Dive** | AWS Core Services, Compute & Serverless, Architecture & Best Practices | EC2, S3, IAM, Lambda, ECS, EKS, API Gateway, Well-Architected Framework, HA/DR, migration strategies |
| **3. Multi-Cloud & IaC** | Microsoft Azure, Google Cloud Platform, Infrastructure as Code | Azure core services, Entra ID, GCP compute & data services, Terraform, CloudFormation, Ansible |
| **4. Cloud Operations** | Monitoring & Observability, Security & Compliance, Capstone Project | CloudWatch, X-Ray, Grafana, OpenTelemetry, WAF, KMS, compliance frameworks, production-grade architecture |

---

## Who This Course Is For

- IT professionals transitioning to cloud roles
- Developers wanting to build cloud-native applications
- System administrators moving from on-premise to cloud
- Solutions architects designing cloud infrastructure
