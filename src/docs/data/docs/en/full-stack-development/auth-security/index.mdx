Authentication is the gateway to every secure web application. This module teaches you how to implement login/register systems, manage sessions and tokens, integrate OAuth providers, and secure your APIs against common attacks.

## Module Pages

| Page | What You'll Learn |
|------|-------------------|
| [Authentication Concepts](/learn/full-stack-development/auth-security/auth-concepts/) | Authentication vs authorization, session-based vs token-based, hashing vs encryption, HTTPS |
| [Password Hashing with bcrypt](/learn/full-stack-development/auth-security/bcrypt/) | Why hash passwords, bcrypt salt rounds, async hashing, comparing passwords, hash storage |
| [JWT (JSON Web Tokens)](/learn/full-stack-development/auth-security/jwt/) | JWT structure, signing (HS256, RS256), access tokens, refresh tokens, token expiry, blacklisting |
| [Session-Based Authentication](/learn/full-stack-development/auth-security/sessions/) | Express sessions, session stores (memory, Redis), cookies, `cookie-parser`, CSRF protection |
| [OAuth 2.0 Integration](/learn/full-stack-development/auth-security/oauth/) | OAuth flows (Authorization Code, PKCE), Google/GitHub login, Passport.js, redirect URIs |
| [CORS & Security Headers](/learn/full-stack-development/auth-security/cors-headers/) | CORS policy, preflight requests, Helmet.js, CSP headers, XSS protection, HSTS |
| [Rate Limiting & Brute Force](/learn/full-stack-development/auth-security/rate-limiting/) | Rate limiting strategies, `express-rate-limit`, account lockout, CAPTCHA integration |
| [Input Validation & Sanitization](/learn/full-stack-development/auth-security/input-validation/) | Server-side validation, `express-validator`, Joi/Zod schemas, SQL/NoSQL injection prevention |
| [Project: Full Auth System](/learn/full-stack-development/auth-security/auth-project/) | Build a complete auth system with register/login/logout, JWT, protected routes, refresh tokens |

## Key Takeaways

By the end of this module, you'll be able to implement secure authentication systems using JWT and sessions, integrate OAuth providers, protect APIs against common attacks, and follow security best practices in your full stack applications.
